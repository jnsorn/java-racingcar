## Commit Message Convention
* ``Feat`` feature
* ``Fix`` bug fix
* ``Docs`` documentation
* ``Style`` formatting, missing semi colons...
* ``Refactor``
* ``Test`` when adding missing test

## Rule for TDD  
1. create fail test code
2. create test as much as possible
